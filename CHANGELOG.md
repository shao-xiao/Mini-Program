# 更新记录

## V1.1.0 - 2026-05-26

### 新增

- 新增统一联系方式校验工具 `ContactValidators.java`，用于后端合同、租户、租户联系人接口兜底校验。
- 新增前端联系方式校验工具 `dehui-property-admin/src/utils/contactValidation.js`，统一 WEB 端联系电话和邮箱校验规则。
- 新增数据库迁移 `V5__tenant_contact_fields.sql`，补齐：
  - `tenant.contact_person`
  - `tenant.contact_email`
  - `tenant_contact.email`

### 修改

- 合同管理 `ContractList.vue`：
  - 新增/编辑合同时校验联系电话、邮箱。
  - 增加合同编辑入口。
  - 增加“恢复”按钮，终止/作废后的合同可以恢复履约。
  - 历史异常联系方式在列表中标记“格式异常”，不自动删除。
- 租户管理 `TenantList.vue`：
  - 租户新增/编辑时校验联系电话、邮箱。
  - 租户联系人账号维护中增加手机号必填校验。
  - 租户联系人账号维护中增加邮箱格式校验。
  - 历史异常联系方式只提示，不清空。
- 后端接口：
  - 合同、租户、租户联系人接口提交错误联系电话或邮箱时返回 HTTP 400。
  - 错误提示统一为 `联系电话格式不正确`、`邮箱格式不正确`。

### 验证

- 后端全量测试：`mvn test`，40 个测试全部通过。
- 前端构建：`npm run build` 通过。
- 后端本地启动正常：`8080`。
- 前端本地启动正常：`5173`。
- 浏览器已回到 `/contracts`。
- 合同、租户、租户联系人接口在错误手机号/邮箱时返回 HTTP 400。

## V1.0.0 - 2026-05-19

### 新增

- 微信小程序前端纳入统一仓库，后台管理前端、Spring Boot 后端、微信小程序三端统一管理。
- 后端健康检查接口 `/api/ping`，用于验证公网域名、Nginx 和后端服务链路。
- 微信小程序公网联调配置，生产接口地址为 `https://wuye.mingda.com.cn/api`。
- 账单审核与来源追踪，支持审核通过、审核驳回、审核备注、审核人、审核时间、来源类型和来源 ID。
- 员工签到能力，支持小程序上班签到、下班签退、定位、备注和签到记录查询。
- 小程序接口联调设置，支持检查后端、切换接口地址、读取开发联调数据。
- 访客车牌号字段，支持小程序提交和后台独立展示。
- 资产台账模块，支持资产新增、编辑、删除、状态变更、转移、维保、操作日志和统计。
- 租户联系人账号管理，支持联系人新增、编辑、停用、重置初始密码和小程序绑定租户联系人。
- 楼宇、楼层、房间字段扩展，为资产定位、租户绑定和后续运营数据打通提供基础字段。

### 修改

- 小程序取消会议、取消访客、取消工单、工单评价改为 `POST`，提高微信端接口兼容性。
- 小程序请求封装改为动态读取接口地址，避免写死单一 `baseURL`。
- 小程序账单页面展示标题、来源、备注等后端新增字段。
- 小程序招商线索提交成功后展示线索 ID，方便后台核对。
- 后台账单列表增强，支持审核状态、来源类型、逾期状态和审核操作。
- 后台财务看板适配账单审核与应收数据。
- 后台访客、会议、招商、工单页面增加小程序来源、申请人类型、车牌号等细节展示。
- 租户管理页面增加租户联系人入口和联系人账号管理操作。
- README 文档补充项目结构、路径标准、部署命令、接口说明和关键改动说明。

### 验证

- 后台前端已在本机执行 `npm run build`，构建通过。
- 后端当前本机没有 `mvn` 命令，需在安装 Maven 的服务器执行：

```bash
mvn clean package -DskipTests

```
